### [#1019 feat: stateless Interactions API transport for Gemini (Phase 1)](https://github.com/allenhutchison/obsidian-gemini/pull/1019)

Phase 1 of the Gemini Interactions API migration (epic #1013). Introduces a new `useInteractionsApi` setting (default **off**) that routes `GeminiClient` through Google's `client.interactions.create` API instead of `generateContent`. The plugin continues to own and replay conversation history (Markdown-backed), so `store: false` is set and no `previous_interaction_id` is used — the `ModelApi` interface and everything above it (agent loop, factory, retry decorator, Ollama provider) are entirely unchanged.

Streaming is not yet supported in this phase: when the flag is enabled, the streaming path falls back to a single-chunk emission via the non-streaming call, honoring the `ModelApi` fallback contract. Real step-based streaming is planned for Phase 2 (#1015). The flag defaults off and `generateContent` is retained as a fallback because the SDK marks `client.interactions` experimental and Google published a "breaking changes May 2026" notice. Fixes #1014.
