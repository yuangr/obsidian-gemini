### [#1046 feat(agent-view): add opt-in Plan Mode UI affordance](https://github.com/allenhutchison/obsidian-gemini/pull/1046)

Implements Plan Mode (fixes #623). A map-icon toggle in the agent view input toolbar (or the "Toggle Plan Mode" command) causes the agent to generate a step-by-step plan with no tool calls, then show Approve & Execute / Reject buttons before acting. On approval the plan is saved to session history as a `[!plan]+` callout and a follow-up execution call is made with full tools. On rejection the request is cancelled and plan mode deactivates. Adds 9 i18n keys, styles for the active-state toggle and plan card, and three new session-history round-trip tests.

### [#1051 ci(i18n): add Update UI translations workflow](https://github.com/allenhutchison/obsidian-gemini/pull/1051)

Adds `.github/workflows/i18n-translate.yml`, a new GitHub Action that automatically regenerates `src/i18n/` from the English source whenever `en.ts` or `translate.mjs` lands on master, on a weekly schedule, or via manual dispatch. The action calls `npm run translate` (incremental — only stale keys), formats output with Prettier, and opens a `chore(i18n)` PR scoped to `src/i18n/`. Clears the existing backlog of ~260 stale keys once dispatched. `AGENTS.md` i18n notes updated to reflect the new automated flow.

### [#1053 feat(agent-loop): streaming hook for follow-up requests](https://github.com/allenhutchison/obsidian-gemini/pull/1053)

Adds `onFollowUpChunk` to `AgentLoopHooks` so follow-up responses after tool batches can stream token-by-token to the UI (fixes #663). When the hook is set and the model API implements `generateStreamingResponse`, the loop switches to streaming for that call; headless callers that don't set the hook continue using the non-streaming path. The agent view wires up `createFollowUpStream` / `finalizeFollowUpStream` to show a live text container during streaming and finalize to proper markdown on completion, preventing a duplicate bubble. Four new tests cover streaming vs non-streaming fallback, thought-only chunk filtering, and multi-iteration accumulation.

### [#1056 fix(rag): rename "Reindex All" → "Rescan Vault" to match actual behaviour](https://github.com/allenhutchison/obsidian-gemini/pull/1056)

Corrects a mislabel that has existed since `abd6619` (flagged by CodeRabbit on #674): the vault scan buttons call `indexVault()` with `smartSync: true`, which skips unchanged files — they scan for changes rather than doing a full reindex. Renames the i18n keys (`settings.rag.reindexButton`, `ragStatus.reindexButton`) and completion notice strings to use "Rescan vault" / "Rescan complete" language. Updates `docs/guide/semantic-search.md` to describe the button accurately and adds a callout explaining that Delete index + Rescan vault is the path for a true full re-embed. No scanner behaviour changes.
