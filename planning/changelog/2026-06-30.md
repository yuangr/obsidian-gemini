## What shipped

### [#1058 feat(ollama): auto-detect vision capability via /api/show probe](https://github.com/allenhutchison/obsidian-gemini/pull/1058)

Replaces the brittle `VISION_NAME_HINTS` name-pattern list with a structured `/api/show` probe, so new vision models are detected automatically without code changes. Detection uses three tiers in order of authority: the `capabilities` array on current Ollama versions, a template regex fallback for older Ollama daemons, and the old name-hint list as a last resort when the daemon is unreachable. The probe response is cached per model name so a future tool-support probe (#709) can reuse it without an extra network call. Fixes #708; produced by the auto-dev pipeline from the approved plan.

### [#1066 refactor: extract openPluginSettingsTab helper for @ts-expect-error sites](https://github.com/allenhutchison/obsidian-gemini/pull/1066)

Architecture-audit nightly sweep flagged three call sites duplicating the same `app.setting.open()` + `app.setting.openTabById(...)` incantation, each behind a `@ts-expect-error` since `App.setting` isn't in Obsidian's typings. Consolidates the workaround into `src/utils/obsidian-settings.ts`, with callers passing `this.manifest.id` instead of a hardcoded plugin-id literal.
