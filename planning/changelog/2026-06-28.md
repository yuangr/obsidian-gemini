### [#1056 fix(rag): rename "Reindex All" → "Rescan Vault" to match actual behaviour](https://github.com/allenhutchison/obsidian-gemini/pull/1056)

The settings panel button and status modal button both previously said "Reindex All" / "Reindex vault," but the underlying call uses `smartSync: true` — it scans for _changed_ files, not a full re-embed. This PR renames the i18n keys to "Rescan vault" and updates the completion notice from `RAG indexing complete: 42 indexed, 18 unchanged` to `Rescan complete: 42 re-indexed, 18 unchanged`. The `docs/guide/semantic-search.md` guide is updated to match and now explicitly calls out that Delete index + Rescan vault is the path for a true full re-embed (e.g., after switching models). No scanner logic changed — only labels and docs.
