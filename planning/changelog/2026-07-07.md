### [#1162 refactor: eliminate all 150 circular imports via a leaf plugin interface](https://github.com/allenhutchison/obsidian-gemini/pull/1162)

Fixes #1155. `madge --circular` reported 150 circular dependency chains in `src/`, almost all routing through `main.ts` (91 files imported the plugin class via `import type ObsidianGemini from '../main'`, while `main.ts` imports nearly the whole codebase). A first attempt at a plugin interface made things worse (150 ŌåÆ 155) because the interface itself had to import ~27 service modules that import the plugin type back. This PR instead uses module augmentation: a new leaf `src/types/plugin.ts` declares only core members, and `src/types/plugin-services.ts` contributes all 32 service/manager handles via `declare module` ŌĆö a file nothing imports, so it can never join a cycle. `main.ts`'s class now `implements` the merged interface, keeping it in sync at compile time. Zero runtime change (all migrated imports were already type-only; `main.js` is byte-identical to master), and `npm run lint:cycles` (new CI gate, `madge --circular`) now enforces the 0-cycle baseline going forward.

This PR also documents the new pattern in `AGENTS.md` (pattern #15: depend on the leaf interface, never `main.ts` directly; put shared manager/sibling types in leaf modules).
